Application Overview

Syllabye is an application created to serve professors and other university administrators in the process of creating and updating syllabi. Syllabi tend to change every semester with new requirements such as updated classroom content, health and safety guidelines, links to campus resources, schedule changes, and many more features. The end goal of the software for the user is to help them make this process less time-consuming so that educators can spend more time devoted to teaching and helping students.

Technology Overview

The technology we used for this application includes:

    Languages
  • HTML and CSS
  • JavaScript
    Platforms
  • Web application
    Hosting
  • Microsoft Azure
  • Github

    Feature List

  • Simple and intuitive UI
  • Convert syllabi to PDF
  • Keep formatting fairly consistent with minimal customization options
  • Store past syllabi in a database to be updated later on

    Future Feature List

  • Login integration with other university applications
  • Use of generative AI technology to save professors more time from writing classroom policies
  • Pull the most recently updated univeristy guidelines automatically to include in the syllabi

    Basic Technical Features

  • Proficiency in creating an intuitive website with HTML, CSS, and JavaScript
  • Deploying and hosting the website with Microsoft Azure and Github
  • Creating server side pages using Node.js and Express to handle additional tasks separately

    Advanced Technical Features

  • Converting from text-based user inputs to PDFs that can be downloaded and printed
  • Adding more customization features such as additional fonts, font-sizes, and custom banners for academic departments
  • Ability to store past syllabi in a database to be easily modified later
